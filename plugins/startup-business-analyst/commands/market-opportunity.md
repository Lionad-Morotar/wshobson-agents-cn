# 市场机会分析

为初创公司生成综合市场机会分析，包括使用自下而上和自上而下方法论计算的总可寻址市场（TAM）、可服务可用市场（SAM）和可获得市场（SOM）。

## 此命令的功能

此命令通过交互式市场规模流程指导用户完成以下步骤：

1. 定义目标市场和客户细分
2. 收集相关市场数据
3. 使用自下而上方法论计算 TAM
4. 使用自上而下分析进行验证
5. 通过适当的筛选条件缩小至 SAM
6. 估算现实的 SOM（3-5 年机会）
7. 以格式化报告呈现发现

## 给 Claude 的说明

当调用此命令时，请遵循以下步骤：

### 第 1 步：收集背景信息

向用户询问关键信息：

- **产品/服务描述：** 解决什么问题？
- **目标客户：** 谁是理想客户？（行业、规模、地理位置）
- **商业模式：** 定价如何运作？（订阅、交易等）
- **阶段：** 公司处于什么阶段？（预发布、种子轮、A 轮）
- **地理范围：** 初始目标市场（美国、北美、全球）

### 第 2 步：激活 market-sizing-analysis 技能

market-sizing-analysis 技能提供全面的方法论。参考它以获取：

- 自下而上计算框架
- 自上而下验证方法
- 特定行业模板
- 数据源推荐

### 第 3 步：进行自下而上分析

**对于 B2B/SaaS：**

1. 定义客户细分（公司规模、行业、用例）
2. 估算每个细分市场的公司数量
3. 确定每个细分的平均合同价值（ACV）
4. 计算 TAM：Σ（细分市场规模 × ACV）

**对于消费者/市场平台：**

1. 定义目标用户人口统计
2. 估算总可寻址用户数
3. 确定每用户平均收入（ARPU）
4. 计算 TAM：总用户数 × ARPU × 频率

**对于交易/电子商务：**

1. 估算总交易量（GMV）
2. 确定抽成率或利润率
3. 计算 TAM：总 GMV × 抽成率

### 第 4 步：收集市场数据

使用可用工具研究：

- **WebSearch：** 查找行业报告、市场规模估算、上市公司数据
- **引用所有来源**并附上 URL 和发布日期
- **清晰记录假设条件**

推荐数据源（来自技能）：

- 政府数据（人口普查、劳工统计局）
- 行业报告（Gartner、Forrester、Statista）
- 上市公司文件（10-K 报告）
- 贸易协会
- 学术研究

### 第 5 步：自上而下验证

验证自下而上计算：

1. 从研究中查找总市场类别规模
2. 应用地理筛选
3. 应用细分/产品筛选
4. 与自下而上 TAM 比较（应在 30% 以内）

如果差异 > 30%，调查并解释差异原因。

### 第 6 步：计算 SAM

应用现实筛选以缩小 TAM：

- **地理：** 实际可服务的区域
- **产品能力：** 服务所需的功能
- **市场准备度：** 准备采用的客户
- **可寻址转化：** 可以触达和转化的客户

公式：

```
SAM = TAM × 地理位置 % × 产品适配度 % × 市场准备度 %
```

### 第 7 步：估算 SOM

计算现实的可获得市场份额：

**保守方法（推荐）：**

- 第 3 年：SAM 的 2-3%
- 第 5 年：SAM 的 4-6%

**考虑因素：**

- 竞争强度
- 可用资源（资金、团队）
- 市场进入效果
- 差异化优势

### 第 8 步：创建市场规模报告

生成包含以下内容的综合 Markdown 报告：

**第 1 部分：执行摘要**

- 一段话概括市场机会
- TAM/SAM/SOM 关键数字

**第 2 部分：市场定义**

- 解决的问题
- 目标客户画像
- 地理范围
- 时间跨度

**第 3 部分：自下而上分析**

- 客户细分分解
- 带来源的细分市场规模
- TAM 计算公式
- 记录的假设条件

**第 4 部分：自上而下验证**

- 行业类别和规模
- 筛选应用
- 验证后的 TAM
- 与自下而上的比较

**第 5 部分：SAM 计算**

- 应用的筛选及其理由
- SAM 公式和结果
- 细分级别的分解

**第 6 部分：SOM 预测**

- 市场份额假设
- 第 3 年和第 5 年估算
- 客户数量含义
- 收入预测

**第 7 部分：市场增长**

- 行业增长率（CAGR）
- 关键增长驱动因素
- 5 年市场演变

**第 8 部分：验证和合理性检查**

- 上市公司比较
- 客户数量验证
- 竞争背景

**第 9 部分：投资论点**

- 市场机会评估
- 主要优势和风险
- 风投规模潜力
- 下一步行动

### 第 9 步：保存报告

提议将报告保存为 Markdown 文件：

- 建议文件名：`market-opportunity-analysis-YYYY-MM-DD.md`
- 使用 Write 工具创建文件
- 与用户确认文件位置

## 获得最佳效果的提示

**应该做：**

- 从自下而上开始（最可信）
- 始终用自上而下进行三角验证
- 引用所有数据源
- 记录每个假设
- 对 SOM 保持保守
- 与上市公司基准进行比较
- 解释任何数据缺口或局限性

**不应该做：**

- 仅依赖自上而下
- 挑选乐观的数据
- 在没有充分理由的情况下声称 >10% 的 SOM
- 不恰当地混合方法论
- 忽略竞争背景
- 跳过验证步骤

## 使用示例

```
User: /market-opportunity

Claude: 我将帮助您创建综合市场机会分析。让我先收集一些背景信息。

您正在分析什么产品或服务？
→ "面向电子商务公司的 AI 邮件营销"

谁是您的目标客户？
→ "北美地区年收入 100 万美元以上的电子商务公司"

您的定价模式是什么？
→ "订阅：根据邮件量每月 50-500 美元，平均每月 300 美元"

[Claude 继续进行分析、收集数据、计算 TAM/SAM/SOM 并生成报告]
```

## 与其他命令的集成

此命令与以下命令配合良好：

- `/financial-projections` - 使用 SOM 构建收入模型
- `/business-case` - 在业务案例中包含市场规模

## 注意事项

- 市场规模分析通常需要 30-60 分钟进行全面分析
- 质量取决于数据可用性 - 解释局限性
- 随着市场发展每年更新
- 保守估算可建立投资者信誉
